🛡️ Podman v5.8.3 Addresses CVE-2026-44517 in Dockerfile Builds

This week, Podman v5.8.3 was released to resolve CVE-2026-44517, which previously allowed unauthorized file inclusion during Dockerfile builds using specific instructions. This update enhances build security by preventing malicious Git repositories or tar archives from accessing files outside the build context.

🚀 Key Releases

Podman v5.8.3
This patch release addresses CVE-2026-44517. The vulnerability involved `ADD` or `COPY` instructions in Dockerfiles potentially including files outside the build context or copying them into the build when interacting with malicious Git repositories or tar archives. The fix prevents this unauthorized file access during the build process.

Dapr v1.18.0
This release introduces optional cryptographic signing and verification for workflow history. When enabled, workflow history is signed and verified on every state load, allowing detection of tampering. This feature enhances the integrity and security of Dapr workflows.

Kuma v2.14.0
Kuma now supports mesh-scoped zone proxies, which allows for per-mesh Zone Ingress/Egress deployments that can be individually targeted by policy. This replaces the previous single zone-wide proxy model. Additionally, sidecar containers are now the default deployment method.

CoreDNS v1.14.4
This version improves transport security for DoH3 and DoQ protocols. It also includes enhancements for DNSSEC signing behavior and adds support for the loong64 architecture. Configurable cache verification and hostname resolution for forward targets were also introduced.

OPA v1.17.1
This release updates the Go version used to build OPA to 1.26.4. This update specifically fixes standard library vulnerabilities (GO-2026-5039, GO-2026-5037) present in code utilized by OPA's HTTP handler and crypto builtins.
