⚠️ Capsule v0.13.0 Changes Default for Webhook Certificates

This week, Capsule v0.13.0 was released, changing the default certificate management for its admission webhooks to use cert-manager self-signed certificates. This alters the previous optional setting to the new default for Capsule deployments.

🚀 Key releases

Capsule v0.13.0 was released, defaulting to cert-manager self-signed certificates for its admission webhooks. This changes the previous optional setting to the new default for certificate management, impacting how Capsule deployments handle webhook security.

Prometheus v3.12.0 introduced security fixes, new features in PromQL and Service Discovery, performance improvements within TSDB, and enhancements to Start Timestamp functionality. This release addresses identified vulnerabilities and expands query and monitoring capabilities.

Karmada v1.18.0 was released, representing a new major version for the multi-cluster Kubernetes orchestration project. The release includes contributions from multiple developers. Patch releases v1.17.3, v1.16.6, and v1.15.9 were also published.

Volcano v1.15.0 was released. According to the maintainers, this version strengthens Volcano as a unified scheduling platform for converged general-purpose and AI computing at scale, supporting batch training, inference, AI Agent, HPC, and big-data workloads.

Skopeo v1.23.0 and Buildah v1.44.0 were released. Both versions include a change in the configuration file lookup behavior. Callers of functions in these projects that read configuration files should refer to the updated documentation for details on resolution.
