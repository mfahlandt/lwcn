⚙️ Kubernetes v1.36 Releases with User Namespaces and Fine-Grained Kubelet API Authorization GA

This week, Kubernetes v1.36 was released, marking several features as Generally Available (GA), including User Namespaces and Fine-Grained Kubelet API Authorization. These advancements enhance security isolation and granular access control within Kubernetes clusters.

🚀 Key releases

Kubernetes v1.36.0
  Introduced User Namespaces to GA, enabling containers to run with non-root user IDs mapped to root within the container, which enhances security isolation. Fine-Grained Kubelet API Authorization also graduated to GA, providing more granular access control for Kubelet APIs. SELinux Volume Label Changes reached GA, improving security by correctly applying SELinux labels on persistent volumes. Gateway API v1.5 moved several features to Stable, indicating API maturity for ingress and egress management.

Envoy v1.38.0
  This release includes breaking changes: the `max_early_data_bytes` field in `tcp_proxy` must now be explicitly set for `upstream_connect_mode` values other than `IMMEDIATE`, and missing configurations will cause startup validation failures. Additionally, the `on_demand` filter no longer performs internal redirections.

cert-manager v1.19.5
  A patch release was published to address reported vulnerabilities. This version includes dependency bumps for Go, which resolve several CVEs.

kyverno v1.16.4
  A patch release fixed CVE-2025-68121. It also included a fix for policy report creation for namespaced policies during admission and background scans.

CoreDNS v1.14.3
  This version adds Windows service support and introduces full TSIG verification across DoH, DoH3, QUIC, and gRPC transports. It also includes improved TSIG propagation, DoH request validation, and optional TLS for the metrics endpoint.
